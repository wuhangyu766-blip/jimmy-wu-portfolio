# 更清晰的玫瑰色循环花朵设计说明

## 目标

让白色主题下的背景花朵更容易被看见，并确保访客能够感知到自然的循环绽放，而不是在首次鼠标移动后永久静止。继续保持 16 片花瓣、整页背景、低资源消耗和不遮挡文字的设计。

## 已确认的根因

当前花朵并非没有运行，而是存在三个叠加问题：

1. 花瓣在约 1 秒内已经接近完全盛开，首次变化很容易错过。
2. 首轮结束后，任意一次 `pointermove` 都会永久删除动画；正常浏览时的轻微鼠标移动就会触发。
3. 白色主题下舞台透明度为 `.68`，外圈花瓣最大透明度为 `.48`，浅粉渐变本身的颜色透明度也偏低，因此最终视觉对比度不足。

## 视觉设计

白色主题改为灰调玫瑰红，不使用高饱和荧光粉：

- 外圈花瓣目标透明度约 `.62`，内圈约 `.72`。
- 花朵舞台透明度提高到约 `.78`；手机端约 `.50`，避免压住正文。
- 花瓣渐变以浅玫瑰粉、灰调莓红和少量透明白组成；边缘使用半透明莓红描边。
- 花瓣纹理仍只使用渐变、描边和轻量内阴影，不加入模糊、背景模糊或大型外阴影。
- 花芯保留暖金色，但将中间色加深，让花芯与花瓣形成更清楚的视觉中心。
- 深色主题继续采用冷调蓝紫花瓣，并适度提高透明度，以保持与深色背景的层次。

## 动画设计

每轮动画固定为 10 秒：

1. 前约 2.8 秒逐片展开。花瓣从更小的折叠状态开始，外圈和内圈通过现有延迟形成自然的错峰绽放。
2. 完全盛开状态保持约 3.5 秒。
3. 随后缓慢收拢，在本轮结束前回到花苞状态并开始下一轮。

动画仍仅改变 `transform` 和 `opacity`。不引入 WebGL、逐帧 JavaScript、`filter: blur()`、`backdrop-filter`、持续 `will-change` 或 3D 位移动画。

## 鼠标交互

首轮 2.8 秒绽放必须完整播放，期间忽略鼠标移动。

首轮结束后：

- 检测到精细指针设备的 `pointermove` 时，在根节点设置 `data-flower-paused="true"`。
- CSS 通过 `animation-play-state: paused` 暂停花瓣和花芯动画，保留鼠标移动当下的自然状态。
- 每次移动都会刷新一个 1.5 秒空闲计时器。
- 鼠标连续静止 1.5 秒后移除暂停标记，动画从原位置继续，而不是从头跳变。
- 触屏设备不注册该监听器。

这样既保留“鼠标动时停止”的反馈，也保证访客之后仍能看到重复绽放。

## 无障碍与降级

- `prefers-reduced-motion: reduce` 下直接显示完全盛开的静态花朵。
- 低动态模式不注册鼠标移动监听器。
- JavaScript 不可用时，CSS 动画仍可正常循环。
- 中英文、深浅色切换、主页信息布局和 GitHub 链接保持不变。

## 测试与验收

自动化测试需要验证：

- 动画时长为 10 秒，花瓣展开阶段约为 2.8 秒。
- 白色主题包含新的玫瑰红渐变、提高后的舞台与花瓣透明度。
- 不再使用永久静态的 `data-flower-static` 作为普通鼠标交互结果。
- 存在 `data-flower-paused="true"`、`animation-play-state: paused`、2.8 秒监听启用延迟和 1.5 秒恢复计时。
- 低动态模式仍为静态盛开。
- 花朵部分继续满足无模糊、无持续 `will-change`、无 `translateZ()` 和无连续漂浮动画。

浏览器验收需要覆盖：

- 白色主题桌面版花瓣比当前版本明显，但不会妨碍姓名、简介和右下重点信息。
- 首轮绽放清晰可见；首轮后移动鼠标会暂停；静止 1.5 秒后继续。
- 390px 手机视口无横向溢出，正文仍清晰可读。
- 英文、深色主题和低动态模式保持正常。
